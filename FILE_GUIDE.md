# Project File Guide

This guide explains every project-owned file currently in this workspace (excluding generated/dependency trees like `.next/` and `node_modules/`).


| File                          | File Type                   | What It Does                                                                                                  | How It Fits                                                                |
| ----------------------------- | --------------------------- | ------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| `/.DS_Store`                  | macOS metadata file         | Finder view metadata for the project folder.                                                                  | Not part of the app; safe to ignore for engineering.                       |
| `/.gitignore`                 | Git ignore config           | Defines files/folders Git should not track (`.next`, `node_modules`, etc.).                                   | Keeps repo clean and avoids committing generated artifacts.                |
| `/README.md`                  | Markdown documentation      | High-level overview of the MVP, setup, architecture, and next steps.                                          | Primary onboarding and project context doc.                                |
| `/app/.DS_Store`              | macOS metadata file         | Finder metadata for the `app` directory.                                                                      | Not part of runtime behavior.                                              |
| `/app/api/recommend/route.ts` | TypeScript API route        | Handles `POST /api/recommend`, validates input, returns computed recommendations.                             | Backend entry point inside Next.js App Router.                             |
| `/app/globals.css`            | CSS stylesheet              | Defines global styles, layout, typography, colors, responsive behavior, and UI components.                    | Shared presentation layer for the whole frontend.                          |
| `/app/layout.tsx`             | TSX React layout            | Root HTML/body wrapper and metadata for the app.                                                              | App Router shell applied to all routes.                                    |
| `/app/page.tsx`               | TSX React page              | Main planner UI: upload photo, draw zones, choose inputs, visualize overlays, view recommendations/calendar.  | Core product experience (MVP flow).                                        |
| `/lib/climate.ts`             | TypeScript module           | ZIP-to-zone heuristic, frost date estimates, month/season helpers.                                            | Climate intelligence utilities used by recommendation logic and UI labels. |
| `/lib/plants.ts`              | TypeScript module           | In-app plant dataset with categories, sun needs, companion/antagonist relationships, care info.               | Domain data source powering recommendations.                               |
| `/lib/recommend.ts`           | TypeScript module           | Recommendation engine: scoring, sun exposure inference, placement generation, and planting calendar assembly. | Core decision engine for intelligent suggestions.                          |
| `/lib/types.ts`               | TypeScript types            | Shared interfaces/types for requests, responses, plants, placements, and zones.                               | Contracts that keep frontend/API/domain logic aligned.                     |
| `/next-env.d.ts`              | TypeScript declaration file | Next.js-generated type references for React/Next environment.                                                 | Required TS integration file for Next.js projects.                         |
| `/next.config.js`             | JavaScript config           | Next.js framework config (`reactStrictMode`, etc.).                                                           | Runtime/build configuration surface for app behavior.                      |
| `/package-lock.json`          | NPM lockfile (JSON)         | Exact dependency tree and resolved versions from install.                                                     | Ensures reproducible installs across machines/environments.                |
| `/package.json`               | NPM manifest (JSON)         | Project metadata, scripts (`dev`, `build`, `typecheck`), dependencies/devDependencies.                        | Defines tooling, runtime dependencies, and commands.                       |
| `/tsconfig.json`              | TypeScript config (JSON)    | Compiler options, path aliases, strictness, include/exclude patterns.                                         | Controls static type-checking rules and TS project behavior.               |
